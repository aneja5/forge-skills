Invoke the forge-skills:spec-driven-development skill.

First check if `.forge/idea-brief.md` exists — read it if so and skip questions already answered.

Interview the user, explore the codebase, design the modules, and write `.forge/prd.md` using
the PRD template in the skill (problem statement, personas, Given/When/Then requirements,
NFRs, data model, module design, release phases, out of scope, open assumptions).

After writing: "PRD written to .forge/prd.md. Run /architect to design the system."
